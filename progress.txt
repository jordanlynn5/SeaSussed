WHAT WE'RE BUILDING
-------------------
A Chrome extension + Cloud Run Python backend that:
1. Opens a persistent side panel when the toolbar icon is clicked
2. Captures a screenshot of any grocery product page on user request
3. Uses Gemini 2.5 Flash (via Google ADK) to identify species, origin, method, certs
4. Scrapes visible related product names from the DOM for alternative scoring
5. Scores the product 0–100 across four sustainability dimensions (pure Python math)
6. Generates a 2–3 sentence explanation via a direct Gemini call
7. Displays grade (A–D), score breakdown, alternatives, and "Not right?" correction

Works on ANY grocery site — Whole Foods, Amazon Fresh, Instacart, and more.
Category: UI Navigator — agent reads the screen visually, not the DOM.

PIPELINE (POST /analyze)
  1. ScreenAnalyzerAgent (ADK LlmAgent, gemini-2.5-flash, vision)
     → extracts species, origin, method, certifications from screenshot
  2. identify_alternative_species() — batch Gemini prompt maps product titles to species
  3. compute_score() — pure Python → ScoreBreakdown + grade
  4. score_alternatives() — scores DOM-scraped or seed alternatives
  5. generate_content() — single Gemini call → explanation + score_factors
  → returns SustainabilityScore

SCORING MODEL (0–100)
  Biological & Population  (max 20): vulnerability (10) + resilience (7) + IUCN (3)
  Fishing/Aquaculture Prax (max 25): gear impact score (wild) or cert+carnivory (farmed)
  Management & Regulation  (max 30): visual cert (15) + NOAA status (10) + FAO exploit (5)
  Environmental/Ecological (max 25): trophic (10) + IUCN (10) + resilience bonus (5)

  Grade: A (80–100) Best Choice | B (60–79) Good Alternative
         C (40–59) Use Caution  | D (0–39)  Avoid

CERT SCORES (for management + practices)
  MSC / ASC                        = 15 pts
  BAP                              = 10 pts
  GlobalG.A.P. / Friend of the Sea = 8 pts
  ASMI / Alaska Seafood            = 7 pts
  Sustainably Sourced / Resp.Farmed = 3 pts
  None visible                     = 0 pts

GEMINI HONESTY RULE (hard rule — no exceptions)
  ScreenAnalyzerAgent: "Extract ONLY what is VISUALLY PRESENT. Return null for
  any field not visible or not determinable with confidence."
  generate_content(): "Explicitly state what was and was not visible on the page.
  Do NOT claim certainty for values Gemini returned as null."

================================================================================
API CONTRACT
================================================================================

GET  /health
     → {"status": "ok", "service": "seasussed-backend", "version": "0.1.0"}

POST /analyze
     Request:  { screenshot: string (base64 PNG), url: string,
                 page_title?: string, related_products?: string[] }
     Response: SustainabilityScore (see below)
     Notes:    full pipeline — vision + scoring + explanation + alternatives

POST /score
     Request:  { product_info: ProductInfo }
     Response: SustainabilityScore
     Notes:    no vision step; used by "Not right?" correction flow

SustainabilityScore response shape:
  {
    score: int (0–100),
    grade: "A"|"B"|"C"|"D",
    breakdown: { biological: float, practices: float, management: float, ecological: float },
    alternatives: [{ species, score, grade, reason, from_page }],  // 1–3 items
    alternatives_label: string,  // "Similar great choices" or "Better alternatives"
    explanation: string,          // 2–3 sentences, states what was visible
    score_factors: [{ category, score, max_score, explanation, tip }],  // 4 items
    product_info: {
      is_seafood: bool,
      species: string|null,
      wild_or_farmed: "wild"|"farmed"|"unknown",
      fishing_method: string|null,
      origin_region: string|null,
      certifications: string[]
    }
  }
